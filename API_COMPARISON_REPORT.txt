## KEY FINDINGS

### High Coverage (>90%)
✓ Most PhotoStack, ImageRef, MetadataRef core APIs exposed
✓ StackManager essentials all exposed
✓ ScanSnapshot pagination fully exposed

### Good Coverage (70-90%)
✓ SearchQuery via JSON serialization
✓ QueryResult in bindings (not in FFI)
✓ Snapshot filtering and status checking

### Low Coverage (<70%)
✗ ImageRef streaming and path access
✗ StackManager event subscriptions
✗ PhotoStack internal setters
✗ QueryResult cursor iteration (core only)

### Extra Features in Bindings
⚠ PhotostaxRepository (.NET only) - single-repo convenience
⚠ CreateSnapshot(ScannerProfile) overload (.NET)
⚠ Promise-based async (TypeScript)
⚠ location() field (TypeScript only)

### Critical Gaps
✗ No snapshot status in TypeScript - photostax_snapshot_check_status exists but not exposed
✗ No ImageRef file path access - cannot determine source file location
✗ No ImageRef deletion - cannot delete image files
✗ No QueryResult status in core FFI - pagination only

### Naming Inconsistencies
- prev_page (core) vs previous_page (bindings)
- clear_handle_caches vs invalidate_caches (different names, same function)
- repo_id not exposed (core has it, no access in bindings)
